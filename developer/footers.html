Basic Footer Guidelines

Every site and landing page we build will have a footer. The footer is generally a place where items with the lowest priority/value will live. The purpose of our site footers is to give legal information, organizational links, social links, and sometimes a call to action or two to help users navigate through the site.

Generally they are very simple to build and shouldn't require much effort, unless the designer is doing something cute like having a fixed element that when the element gets to the footer it locks in place above the footer and continues allowing the user to scroll past it. Other than that scenario footers are generally easy.

Markup For The Footer

Below is a list of semantic markup that should be used in footers to create the layout.

<Footer>
this tag should wrap the entire contents of the footer
<nav>
this tag should be used to wrap each navigation found in the header
<ul>
use unordered lists to wrap the navigations children, this is one place where an extra dom element is acceptable, because it provides meaningful information to users on screen reads and provides additional structure
<section>
any sections in the footer should be wrapped in section tags
<form>
any forms in the footer should be wrapped in form tags
