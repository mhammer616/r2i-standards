Basic Typography Guidelines

Designers spend a lot of time making sure the right fonts are paired together and the sizes work well with the design. It's our job to make sure when those fonts are placed on the page the pairings work well at all sizes. There are different approaches that can be taken from choosing which font-size to use at different media-queries to alter the font size. We'll review some basic guideliness that should be followed while maintaining the designers intent.

Measuring Font-Sizes

We have many different ways to write our CSS measurements: pixels, points, ems, rems, viewport, percentage, inches, centimeters, etc... Why limit ourselves to just using one on the whole website? Before diving into how to choose which measurement to use, lets first get a good understanding of what each measurement type means.

Pixels (px)
This is an absolute measurement of related to pixels on the screen, 1px = 1pixel of the monitor.
Points (pt)
This is an absolute measurement used for print, there are 72 points in an inch.
Inches (in), Centimeters (cm), Millimeters (mm)
These are all absolute forms of measurement, they really shouldn't be used in CSS.
Percentage (%)
This is a relative form of measurement that is based on the parent element.
EM (em)
Relatvie to the font-size of the parent element, a font-size of 1.5 em is 1.5 times the size of the parent elements font-size.
REM (rem)
Releative to the Root Elements font-size, coupled with resetting the font-size of the body, or html tag to 62.5%, means everything is base 10 (on normal browser settings) and you can use REM to do things like 1.6 = 16px.
Viewport Height (vh), Viewport Width (vw)
Relative measurement where 1vh = 1% of the viewports height, and 1vw = 1% of the viewports width.

Now that we understand what each measurement means, here's a list to help choose which type of measurement you should use.

  • Hero headings and text - use viewport measurements so the scale will stay consistent, use media queries to set minimum and maximum font-sizes in rems.
  • Body copy - start by setting body {font-size: 62.5%;}, then use rems to set headings, paragraphs, etc font-sizes.
  • Line Heights - these should be set in ems related to the font-size set for the body copy, typical line-heights are 1.5em or 2em.
  • Padding and Margins - these can be set in REMs or EMs depending on if you want it to be relative to the root element or the element itself.

Determining Font-Sizes for Responsive

Line-length is the most important part for determining font-sizes on a web-site, while the designers set this up for large desktops, it's on the Front End Developer to make sure the site maintains readability throughout the build. Optimal line-length is bestween 45 to 75 characters for body copy. Ideally the number is 66 characters. This helps the reader be able to maintain their line when reading the copy.

Mobile sites should be on the lower end of that line-length, usually being somewhere between 45-55 characters per line, while large devices shouldn't exceed 75 characters.

To acheive optimal line-length use media-queries to set font-sizes that are flexible and help achieve optimal line-length, but don't go overboard creating code bloat. Adjusting at the macro media-queries should be substantial.

Lock in our max-width for the main article to something between 1440px and 1600px should also help with achieveing an optimal line-length.

Bookmarklet is a tool that can help determine line-length and highlights characters 45-75 in any copy you want to test. This tool should only be used when testing, and never be enabled on a production site.

Optimal line-length's don't apply to large format fonts used in headings, although they shouldn't look bad when they line-break either.

Text Alignments

Most of your copy on the site should be left aligned, while headers and large text may break this rule, body copy should be left aligned to make things easier to read.

You options for the text-align property are left, right, center, and justify, while some people like reading and having nice flush ending points for their lines of copy that justify provides, the jagged edge that left provides actually increases readibility. While center creates too much disruption with jagged edges on both sides of the block of text. Right should be used when building a website for rtl languages or aligning something in an element, like a button or navigation.

Heading Elements

Heading elements are used to define importance of text, with the <h1> being the most important heading on the page, while the <h6> is the least important heading.

Headings should be used to style text, as well as create an outline of the page, each section of copy with equal importance should have a heading associated with it that defines it's importance in the copy heirarchy.

Heading 1 <h1>

The <h1> should only be used once on the page, this is the most important heading and is the title of the page. Before determining what copy should be in the <h1> check with SEO for their recommendation. Depending on how the client structures their copy and the design of the site, the most important heading may reside in the page banner or as the first line of text. In addition to this, you may also need muliple banners that allow for the <h1> to be located out of the page banner only on certain pages like News, Events, and Blog detail pages.

Since there should only be one of these on a page, use parent elements to target styling of the h1 in CSS. There should almost never be a need for a class on an h1 other than to set colors if the client has control to do so through options in the CMS.

Headings 2 <h2> - 4 <h4>

<h2> through <h4> should be used primarily in the body copy to separate sections of copy. These can appear multiple times on the page and provide both structure and SEO value to body copy.

Headings 5 <h5> & 6 <h6>

<h5> and <h6> tags can be used for styling content as they have little impact with SEO value. These are commonly used in asides, navs, and footer elements to provide styling capabilities without the use of classnames.

Block Elements

Block elements are elements that are display: block; by default in CSS and unless you set properties on them will create new lines of copy.

Paragraph

This ones obvious so not really going to go into any real detail, if you don't know what a paragraph is click here.

Blockquote

The blockquote is used to show quoted text, commonly used for client/customer testimonials, but should also be styled out for use within text to show a quote from source.

You can also use the cite attribute to provide context in conjunction with the blockquote, if your attribution line is simple enough, the attribute could be used for content in the after to add the attribution line to your blockquote.

Figure

One of the new HTML5 tags, figure, allows you to add context to information that is supplemental to the content.

This tag has two parts, the figure where an image, video, audio, or table can be inserted, then a figcaption to add a caption underneath the figure.

These can be used to style and call out various pieces of supplemental information.

Div

Divs should be used for any element that doesn't fall into the above contexts.

Adding classes to div becomes a necessity if they are used multiple times in the same element.

Line Break

<br /> can be used to insert a new line of text without causing additional spacing from margins and padding.

They should not be used to apply two line breaks to create a space for a paragraph of text, that should be a closing and new p tag.

Horizontal Rule

The hr tag can be used to create lines separating blocks of copy.

List Elements

While lists are block elements, they have additional elements, so we separated them from normal block related content elements.

Unordered List

The <ul> element should be used to create lists where the order doesn't matter to the context of the list. Each item is either of the similar importance or don't involve the previous item to be read first.

The <ul> element wraps it's child elements using <li> tags.

Ordered List

The <ol> element should be used to create lists where the order does matter to the context of the list. The order of each child element is important to the context of the list.

The <ol> element wraps it's child elements using <li> tags.

Definition List

The <dl> element should be used to create lists where a definition is being given.

The <dl> element wraps it's child elements using <dt> for the term and <dd> for the definitions, multiple definitions can be given to a single term. Both the dt and dd are direct children of the wrapping dl tag.

Inline Elements

Inline elements are used mainly to style and provide context to content.

Anchors

Anchors <a> are used to create hyperlinks, either to external pages or internal id's.

When linking to an external website, anything other than the domain of the website you are building, you should open a new tab, or window, by using the target="_blank" attribute.

Bold

To bold text within a block element, apply the <strong> tag to the text.

Even though the <b> tag will do the same thing, it is deprecated from the HTML spec and should not be used anymore.

Italics

To italicize text within a block element, apply the <em> tag to the text.

Even though the <i> tag will do the same thing, it is deprecated from the HTML spec and should not be used anymore.

Underline

To underline text within a block element, apply the <u> tag to the text.

Be careful when using the underline tag as it could cause confusion, a lot of websites use an underline to signify clickable text on a page.

Small

To make smaller text within a block element, apply the <small> tag to the text.

This tag can be helpful in a Hero to supply a second line of text, while keeping it relative to the parent element.

Address

Wrap addresses in the <address> tag to supply context to the element and signifying to the browser or SEO crawler that this is an address.

The tag can also be helpful to style addresses in differently in the content.

Span

When you need to do something else that's not listed above, use a <span> tag to wrap your element and apply a class to it.

Media Elements

Media elements are used to provide visual or audio information such as images, videos, or audio clips.

Images

Images should be added easier standalone as an <img /> tag or wrapped within a figure tag. If the image you are adding has a caption use the figure tag to wrap it and make it easier, if not and it's just an image add the <img /> tag directly where needed.

Adding classes for your different types of figures may be helpful, just keep them contextual to their meaning, if you need figures for videos, images, and tables, then use those as your class name to properly style each look out.

Properly set alt attribute for ALL images, if the image is completely meaningless then the alt attribute should be left blank, otherwise it should be meaningful text to help visually impaired users to understand what the image is. If there is text in the image, the alt attribute should match the text.

Video

Using HTML 5 Video tags can be helpful when you want to add a video to your website. These require the use of two certain types of video files mp4, and ogg/ogv.

The default player doesn't come with any buttons, so you'll have to create your own player tools (possibly will build out a tool for us to use if the need arises). With that being said, it's much easier to use a third party video service, most commonly we suggest YouTube or Vimeo.

Generally, we should wrap our videos in a wrapping element that has position: relative; padding-top: 56.25%; then on either the iframe from a third party video source or video if we are hosting gets position: absolute; left: 0; top: 0; width: 100%; height: 100%; to make the video responsive.

Use data-aspectratio to set the size in CSS of your video wrapping element.
