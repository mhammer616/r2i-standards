Button Guidelines

Buttons are extremley essential to every website we build. The only problem is that button class naming conventions can become messy and inconsistent. In order to standardize our naming system, follow the class naming conventions and code examples below.

Button Styles

There is no more need to define the btn and btn style class anymore, like so class="btn btn-primary". Our new set of css styling pre-defines that button structure via the css style [class*="btn-"]. All you'll need is to use the btn class style to define the type of button you need to insert.

Base Style

When using any button style class, the base button style gets inherited to that button. Just make sure to change the variables in the less file to match the colors, fonts & spacing to the design.
