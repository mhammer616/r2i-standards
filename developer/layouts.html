Basic Layout Guidelines

Every page we create will most likely be able to fall into one of the following layouts we've defined below. Follow the markup and patterns defined to layout your page.

When you get the design, think of the rows and columns needed to create the page. Define the areas into rows and columns that you will place components into that will add content to your page. Use the appropriate Semantic tag to define the columns and elements throughout the page. If a semantic element does not exist, then use a div to define it.

HTML5 Semantic Markup Guide

The use of semantic markup will cut down on classnames and ids used to define areas for the page layout. Below is a list of semantic tags that should be used to define our layouts and what they should be used for.

The use of semantic markup will help define areas of content for screenreaders and aria enabled browsers to make the markup more accessible for people that need assistance.

<header>
This tag should wrap the site header used on every page of the site as a direct child of either the body, or the outer most element that we have access to in the CMS
This tag can also wrap the header within the main content of the site giving additional semantic meaning to things like banners
<footer>
This tag should wrap the site footer used on every page of the site as a direct child of either the body, or the outer most element that we have access to in the CMS
This tag can also wrap the footer of an article or section to add semantic meaning to things that are less meaningful to the article or section
<main>
This should appear only once on a page and should wrap the bulk of the content on the site
This element should be placed after the header and before the footer at the root level, or as close as we have access to, of the page
<nav>
This should wrap every navigational list or area on the page
<aside>
This should wrap the sidebar element on the page
<section>
This can be used to define a group of containing piese of content that are related to each other
<article>
This should wrap a group of content that can stand on it's own independent from the rest of the page
