Basic Tables Guidelines

Tables should only be used to display tabled data. They should never be used for layout purposes.

Tables are used to form data that has rows and columns of information that should be scannable.

Tables are made up of a group of tags:

<table>
This is used to wrap all table elements.
<thead>
This is optional, though it should be used more often than not, and is used to wrap the table row elements that make up the head of the table.
<tbody>
This is required of all tables, regardless of whether or not your table uses a thead or tfoot element, and is used to wrap the body of the table.
<tfoot>
This is optional and is used to wrap the footer of the table.
<tr>
This is used to create a row containing either, or both <td> and <th> elements.
<th>
This is used to create a table cell that is designated as a heading.
<td>
This is used to create a table cell.

Responsive Tables

I'm just going to go ahead and say it... tables suck especially on mobile devices. There's really no good solution for building a usable table on a mobile device and keeping it readable and usable. We try to spin things positively but when it comes to tables on devices, there is no way to do it.

There are really three options when building tables for a responsive website, each with their own benefits and problems...

  1. Wrap the element in a div that has overflow: auto turned on and then when the table becomes too big for the area, a scrollbar will appear so the user can scroll to see more data. Obviously this is less than ideal user experience, but the table still looks and feels like a table, and let's be honest - if a user is trying to read a massive table of data they are probably on a device that it will work on.
  2. Change the table into a list using CSS, or if needed, JS, to display it on the mobile device. The CSS approach will only work on tables that have their data formatted horizontally instead of vertically. The JS approach would be necessary for vertically formatted tables, and should be used over duplicating the content and using hide and show media queries because of SEO reasons.
  3. Hide the table from mobile users, and tell them to use a larger browser to view the table. While this is the easiest it shouldn't be done without the client suggesting to do it this way, or at least approving this first. I would recommend you do approach 1 and add a note below the table stating for optimal display, use a browser larger than xxx.

Bootstrap has a lot of good stuff they do with tables, but, they are catering to the masses with their framework and trying to make something that's suitable for everyone's website. Most likely, you have a couple tables on your website that all look the same, and you may have one additional special table that shows some sales chart, but that's what a class is for. Style the table element for the most common look on your website, then add a class for the special case.

Below in the examples section, you will find examples of each of these approaches, as well as some basic styling of tables.

Data Tables

DataTables is a jQuery plugin that will create searchable, sortable, and filterable, tables with pagination.

This plugin should be used when large amounts of data are displayed with the need for any or all of the above features. There is no use to re-write this plugin as it does a lot of complex things and has a really good website full of documentation and examples.

To see examples, visit the links above. There's no reason to put that much effort when they do a really good job of showing it.
