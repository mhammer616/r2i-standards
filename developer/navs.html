Basic Nav Guidelines

Every site that we build has at least one navigation somewhere on them. Navs are probably one of the most time consuming things to style out on the website. It's also one of the places where we notice the most code bloat, between unnecessary wrapping elements and JavaScript to make a simple hover effect, it can get bloated fast.

With that in mind, keep things simple, keep the markup simple, keep the CSS simple, and try only to use JS for the mobile version where you might be adding things for accordion or expansion functionality.

At the most basic level, navigations are just lists that help the user get around the site quickly. By keeping the markup simple and list based it should make the CSS simpler, while maintaining flexibility for creative designs.

When building the navigation, start with the mobile version and work your way up through the tablet and desktop. Remember touch for both sizing and functionality, and mouse interactions.

Markup For The Nav

Below is a list of semantic markup that should be used in headers to create the layout.

<nav>
this tag should be used to wrap each navigation
<ul>
use unordered lists to wrap the navigations children, this is one place where an extra dom element is acceptable, because it provides meaningful information to users on screen reads and provides additional structure
<li>
wrap inidividual items in a list item, creating hierarchy for each level
<div>
if needed for design purposes wrap child ul elements in an additional div for dropdowns

If the markup is structured properly there should be no need for additional classes for navigations, use wrapping semantic elements as the selector, if that's not enough and you do need a class, use a single on the nav element. The class should be related to the type of navigation it is, acceptable names: main, int, breadcrumb, footer, and utility.
