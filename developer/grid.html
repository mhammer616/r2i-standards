Basic Grid Guidelines

All our websites are built using a Grid. Generally the designers use a 12 Column Base for their grid and stick with that. Instead of using Bootstraps 12 Column Grid we took the idea and re-worked it to meet our needs.

Our grid is designed to work in a wrapping element similar to how Bootstrap does there, the real difference is that we can spin off custom grids and do 8, 10, or 100 column grids with our new grid layout. The reason for this is to give our designers some flexibility, while maintaining a solid base for the grid systems.

When To Use Our Grid

Since our standard approach is use as little classes and extra elements as possible, try to avoid using these for page layout purpose, and more for content or grid layouts within the page. If your nav page element and main page element are 3 columns and 9 columns on large sizes but 4 columns and 8 columns on small and medium, just use media queries in the css to set these. However for something like a form where you want the fields to be lined up in a grid and have them change based on browser size, use the grid.

Some examples of when to use the grid:

  • Form Layouts
  • Article Listing Layouts
  • Image or Video Gallery Layouts
  • Profiles

How To Use Our Grid Out of The Box

Our standard grid is for all browser sizes to have 12 columns, this should work for most websites we design and build. An equal padding of 15px is applied to the left and right side of the grid.

In the boilerplate variable.less file you will find two grid related variables. @gutter defines the amount of padding applied to each colum, it will also apply (-1 * @gutter) to the wrapping row. @colCount is used to define how many columns should be in your grid.

These variables should be altered if the design changes either the Gutter or the amount of columns you need.

Offsets and Helper Classes

Just like in bootstrap we have added offsets and a few helper classes that will assist in the use of our grid.

Offsets

The offsets are done exactly as they are in bootstrap, with adding col-sm-offset-## to the col-sm- class to offset a certain number of columns.

Helpers

There are occasions where you'll want to change the float to rearrange content/structure for visual importance. Using col-[size]-push-[columncount] or col-[size]-push-[columncount] you can change the positioning of the elements. Push indicates the element goes to the right from the left, and pull indicatees it should go to the left from the right. The column count is replaced with the amount of columns it should be moved.

In addition to these we've also added just pull-left and push-right classes which float left and right respectively.

Dive Into The Code

Let's take a look at how the grid is created, and how you can modify this to create new grids for the site.

The Variables

There are two variables used to define items, the first is @colCount which is used later to define how many columns your grid will have. @gutter will be used to define how much padding is applied to the left and right of each column

Use these variables to alter the padding or column count for the grid the designer used.

The Loop

Using Less' Loop function we can create a loop that will create all our classes needed for the columns on the fly, also doing the math to figure out the percentage width needed for that class.

This is done by creating a function .generate-columns() that will be iterated over using the count @i: 1 until it hits our limit @n.

When we initiate our .generate-columns() function we'll call .generate-columns(@colCount). This will run the loop and itterate over where ever we called it adding 1 to whatever @colCount is set to. After writing out our class name it will then take the iterative count multiply that by 100% then divide it by our n variable. After that's complete it will run it again adding 1 to the iterative count. This function will run until it reaches the limit defined by @n.

Calling the Loop

Calling the loop within each mediaquery will trigger Less to preform the loop function we've defined.

For example the col-xs- will output the class for col-xs-1 through col-xs-12 with the width property properly defined.

Create Custom Grids

We can use the above generic setup that is in the boilerplate package, to then create additional grid systems as needed.

To customize the grid for your needs you must first figure out what custom grids you will need. Figure this out by determining how many columns are used in the design.

Some designers may want to do a 8 or 10 for a particular design or browser size.

When this happens, do one of three things: change the variable @colCount to the appropriate size if all the grids are the same; change the variable for the particular size you want to have the unique grids; create a new grid class. Examples of each method can be found below.
