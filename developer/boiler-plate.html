Boilerplate Guidelines

The boilerplate is meant to help get the folder structure setup for resources and give you a base set of files to use as a starting point.

The boilerplate downloads below give you the following files:

  • Less Files and Folders
    • base/base.less - this is where you will load everything needed for the site
    • base/mixins.less - a set of mixins that should be used when developing the site
    • base/variables.less - a set of variables that should be used when developing the site
    • base/resets.less - a set of resets and classes that should be used when developing the site
    • framework/grid.less - the file used to generate the grid and everything that goes with it, documentation found here
    • framework/header.less - a file that should be used for the header of the website
    • framework/footer.less - a file that should be used for the footer of the website
    • framework/layouts.less - a file that should be used for the layouts of the website
    • components/component - a base folder that contains a base.less file for you to build out each component used throughout the site.
  • JS Files
    • base.js - this is where you will put js that is used throughout the site
    • components/component - a base folder for you to copy for each component used throughout the site.

Static Site Boilerplate

The static site boilerplate is the most generic and can be used when building a landing page or site that does not have a CMS behind it.

Download Static Site Boilerplate

Adobe Boilerplate

The Adobe boilerplate has a separate section for components. The individual components are hosted with their specific js/less files under the apps folder. All global css/less files are kept in the clientlib-site folder in the etc directory.

Download AEM Boilerplate

Sitecore Boilerplate

The sitecore boilerplate is basically a duplicate of the Static Site Boilerplate, and should be layered on top of the build when you get started.

Download Sitecore Boilerplate

DNN Boilerplate

The DNN boilerplate is modified from the static site boilerplate to include a Skin and Container folder that should be renamed to match your site. The Container folder will contain blank, and h1-h5 containers. The Skin folder will contain a home skin file, includes (header, and footer), and navs.

Download DNN Boilerplate

Drupal Boilerplate

The Drupal boilerplate is modified fromt he static site boilerplate, to include some drupal speicif files & paths.

To complete proper implmentation, place the resrouces folder into your child theme folder and follow the steps in the Drupal Guidelines.

Download Drupal Boilerplate
