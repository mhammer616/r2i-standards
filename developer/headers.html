Basic Header Guidelines

Every website we build will no doubt have a site header. It may be at its simplest. just holding the logo, or at its most complex, housing the logo, a utility navigation, login form, and main navigation. Most likely the header will be either fixed or relative positioned at the top of the page, but sometimes the header could be fixed to the left of the page. Whatever way the designers layout the header, the more complex headers can be one of the most time consuming pieces we build for the whole project.

For responsive needs, our approach to headers and its contents is not to have multiple for mobile devices or hide and show elements. Write your code in a way that elements can be positioned and moved to the correct location for various screen sizes. Start with the mobile header to get your code structure in place, and move up through each macro media query adjusting font-sizes, padding, heights, and widths as needed. Everything from logo to the utility and main navs should only appear once in the dom.

Markup For The Header

Below is a list of semantic markup that should be used in headers to create the layout.

<header>
This tag should wrap the entire contents of the header
<nav>
This tag should be used to wrap each navigation found in the header
<ul>
Use unordered lists to wrap the navigations children, this is one place where an extra dom element is acceptable, because it provides meaningful information to users on screen reads and provides additional structure
<form>
Any forms in the header should be wrapped in form tags
